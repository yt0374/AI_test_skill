### 分拣系统差异（基于安吉纳分拣系统用户手册）

> 来源: `安吉纳分拣系统用户手册.docx`

#### 任务生命周期（安吉纳特有）

与主线"父任务→拆子任务"模式不同，安吉纳有独立的分拣任务状态机：

```
新增任务(勾选缝制任务单) → 产前准备 → [编辑+分配站位+上线] → [发布] → 待分拣
→ [开始分拣] → 分拣中 → (下架执行) → 完成
```

关键区别:
- **保存 vs 上线**: 保存仅存配置不生效; 上线=保存配置并正式生效
- 分拣任务直接从**缝制任务单**中选择创建，而非来自包装站装箱清单
- 无明确的父任务/子任务拆分概念

#### 上架位操作流程（安吉纳特有细节）

```
1. PAD绑定上架站位（实施阶段完成）
2. 操作工把有衣服的衣架挂到分拣线上架位置
3. 在PAD界面选择生产订单号
4. 选择某个SKU
5. 按下分拣线上的衣架提升操作按钮（物理按钮）
6. 衣架自动提升，分拣条码枪扫描衣架条码进行绑定
7. 显示: 分拣衣架号, 日产量+1, 已上架数+1
```

#### 下架位操作流程（安吉纳特有细节）

```
1. PAD绑定下架站位
2. 点击某任务的【查看】，列出可呼叫SKU（颜色/尺码/已分配数/杆内库存数/可呼叫数）
3. 选择SKU，填写调货数量
4. 点击【呼叫】→【确定】
5. 任务状态: 等待中 → 执行中 → 完成(信息消失)
6. 异常处理: 可【取消】或【手动完成】
7. 辅助功能: 颜色/尺码查询, 可呼叫数开关, 自动刷新时间设置, 【更多】查看详情
```

#### 站位调度状态机

```
新增(空白行, 填写调度/目标站位+衣架数) → 保存 → 初始化
→ [启动] → 进行中 → [完成] → 已完成
```

调度类型: 上架位→库区, 上架位→下架位, 库区→库区, 库区→下架位
