## 1. 硬件定义

### 1.1 硬件类型与编码规则

| 硬件类型 | 编码规则 | 逻辑含义 | 说明 |
|---------|---------|---------|------|
| **分流阀** | Vxxx | = 入站站位 | 衣服进入存储区前的物理分流点，是进站的物理入口 |
| **上架口** | Rxxx | 上架入口 | 衣服首次进入系统的操作口 |
| **下架口** | Pxxx | 下架出口 | 衣服离开存储系统的操作口 |
| **支轨** | Bxxx | 分支轨道 | 连接主线与库区的分支路径 |
| **库区** | Cx'x'x | = 出站站位 | 衣服的存储/缓存区域，按级别分为一级、二级 |
| **电动线** | Sxxx | 一条道 | 单通道主线传输（对应"一条道"概念） |
| **分拣线** | Sxxx | 多条岔路口 | 多分支主轨道（对应"多条岔路口"概念） |

### 1.2 关键硬件区分

- **电动线** vs **分拣线**: 两者均编码为 Sxxx，但前者是单通道，后者是多分支
- **分流阀 (Vxxx)**: 入站站位，是路由决策的物理执行点
- **库区 (Cxxx)**: 出站站位，分为一级库区（混合存储）和二级库区（SKU精准存储）
- **上架口 (Rxxx)** vs **下架口 (Pxxx)**: 分别是系统的入口和出口

## 2. 整体业务流程

### 2.1 主流程（端到端）

```
上架(选SKU) → 分流阀入站 → 库区入口【扫描衣架条码→确认入库】→ 库区杆等待 → 下架呼叫 → 衣架出库 → 下架口【扫描衣架条码→确认出库】→ 包装 → 空衣架回流
```

### 2.3 流程步骤详解

| Step | 名称 | 动作 | 关键数据 |
|------|------|------|---------|
| Step 1 | 上架 | 操作员在工业平板选择SKU，或用扫码枪/RFID扫描唯一码，衣服通过上架口(Rxxx)进入系统 | SKU, 唯一码, 上架口编号 |
| Step 2 | 进站 | 衣架经电动线/分拣线到达分流阀(Vxxx)，此处决定路由方向 | 分流阀编号，路由目标 |
| Step 3 | 库区分配 | 系统判断库区状态：有空→按维度分配；无空→混色混码，进最少杆子 | 库区状态，分拣维度(SKU/分色分码) |
| Step 4 | 等待呼叫 | 衣架在库区杆上静置等待，期间可被包装站或加工区下架任务匹配 | 杆位信息 |
| Step 5 | 任务拆分 | 包装站下发装箱清单→系统生成父任务→拆分为子任务 | 父任务ID，子任务ID，超时计时器 |
| Step 6 | 出站 | 衣架从库区杆滑出→经轨道到达下架口(Pxxx)→操作员取下衣服 | 下架口编号，衣架条码 |
| Step 7 | 空衣架回流 | 空衣架沿轨道返回上架位 | |

## 3. 业务场景详解

### 场景一：正常路径

```
上架 → 一级库区 → 下架 → 空衣架回流
```

**流程**:
1. 操作员在工业平板选择SKU 或 用扫码枪/RFID扫描唯一码，衣服通过上架口(Rxxx)进入系统
2. 衣架经电动线/分拣线到达分流阀(Vxxx)，分流阀=入站站位，此处决定路由方向
3. 系统判断库区状态：
   - 有空闲杆子 → 根据分拣维度(SKU/分色分码)分配
   - 无空闲杆子 → 混色混码，进最少的杆子
   - 分配结果：衣架被路由到目标库区(Cxxx)的指定杆
4. 衣架在库区杆上静置等待，期间可被包装站或加工区的下架任务匹配
5. 包装站下发装箱清单 → 系统生成父任务 → 拆分为子任务
6. 衣架从库区杆滑出 → 经轨道到达下架口(Pxxx)，操作员取下衣服，完成下架
7. 空衣架回流到上架位

### 场景二：干燥房路径（荆门）

```
上架 → 入站(分流阀) → 系统判定目标为干燥房 → 衣架直接路由到干燥房（跳过一级库区判断）
→ 静置计时开始 → 满足全局静置时间 → 下架口可呼叫 → 下架位出站
```

**关键点**:
- 跳过一级库区的容量判断，直接路由到干燥房
- 干燥房有空 → 静置计时（全局/方案级别可配）
- 干燥房满 → 降级一级（降级策略TBD）

### 场景三：缓存杆路径

```
上架 → 入站 → 系统判定目标为缓存杆 → 衣架进入缓存杆 → 锁定SKU
→ 等待正式下架需求 → 下架呼叫触发 → 下架位出站
```

**关键点**:
- 缓存杆类似储备站，暂不出库
- 锁定SKU，保证后续同SKU衣服的一致性
- 需要正式下架需求才能触发下架

### 场景四：插单款快速通道

**新插单款（尚未进缓存杆）**:
```
插单款标识 → 上架 → 入站 → 跳过【缓存杆】→ 直接走快速通道 → 库区分配（一级/二级）→ 优先出库到包装站
```

**旧插单款（已在缓存杆中）**:
```
插单款标识 → 已在缓存杆中的衣服 → 标记为"需优先出" → 下一次下架呼叫时优先匹配 → 出库到包装站
```

**关键点**:
- 新插单款：跳过缓存杆，走快速通道
- 旧插单款：标记优先出，不立即移动，等下次呼叫时优先
- 区分"尚未进缓存杆"和"已在缓存杆中"两种情况

### 场景五：空衣架回流

```
衣服被取下 → 空衣架沿轨道返回 → 入站(分流阀) → 分配到空衣架下架口
```

**关键点**:
- 空衣架回收站策略：少的进→满站也进
- 无视衣架状态（不管衣架是否有问题）
- 回流入站也是通过分流阀

### 场景六：同款"让道"流程

```
衣服入库到一级库区 → 系统判定需让道 → 衣服从一级库区移出 → 沿轨道重新入站
→ 分配到二级库区 → 在二级库区等待下架
（如果二级库区没有空位，就会回流到一级库区）
```

**测试流程**:
1. 先造一批不需要的尺码
2. 然后再造需要的尺码
3. 呼叫下架需要的尺码
4. 前面不需要的尺码就需要让道
5. 如果二级库区有空位，就去二级库区
6. 如果二级库区满了，就回流到一级库区

**验证点**: 需要查看让道衣架的衣架路径是否正确

### 场景七：异常站路由流程

```
衣服入站 → 路由计算 → 发现无法确定目标 → 送入异常站下架口 → 人工处理
```

**触发条件**:
- 未绑定加工方案
- 加工方案已下线
- 未绑定下架任务
- 目标满站

**人工处理**: 跨多级扫描枪的手工调度任务
- 例如：一级库区 → 一级库区
- 例如：一级库区 → 下架位

## 4. 状态机与状态转换

### 4.1 衣架生命周期状态

```
[未入系统]
    │
    ▼
上架扫描 (Rxxx入口，扫描衣架条码→确认入库)
    │
    ▼
[路由中] ──分流阀(Vxxx)决策──
    │           │           │
    ▼           ▼           ▼
插单款路径   干燥房路径   正常路径
    │           │           │
    ▼           ▼           ▼
[快速通道]  [静置计时]  [库区杆等待]
    │           │           │
    └───────────┴───────────┘
                │
                ▼
        [等待下架呼叫]
                │
                ▼
        [下架执行中]
                │
                ▼
        下架扫描 (Pxxx出口，扫描衣架条码→确认出库)
                │
        ┌───────┴───────┐
        ▼               ▼
   [衣服→包装]    [空衣架→回流]
                        │
                        ▼
                  [回到上架位]
```

### 4.2 库区状态

| 状态 | 含义 | 转换条件 |
|------|------|---------|
| 空闲 | 杆子无衣服 | 衣服出站后 |
| 占用(按维度) | 杆子有衣服，按SKU/分色分码维度 | 有空杆子且分配成功 |
| 占用(混色混码) | 杆子有衣服，无空闲杆时混放 | 无空闲杆子 |
| 锁定 | 缓存杆锁定特定SKU | 系统判定为缓存杆 |

### 4.3 任务状态

| 状态 | 含义 | 转换条件 |
|------|------|---------|
| 待拆分 | 父任务已生成 | 包装站下发装箱清单 |
| 执行中 | 子任务正在执行 | 父任务拆分完成，子任务被分配 |
| 超时 | 子任务超过10分钟 | 超时计时器达到10min |
| 完成 | 子任务执行成功 | 衣架成功出站 |

### 4.4 子任务执行约束

| 约束项 | 规则 |
|--------|------|
| 存储杆并发 | 1个存储杆同时只1个任务 |
| 分拣区域并发 | 1个分拣区域同时只1个存储杆执行 |
| 超时 | 子任务超时10min |
| 重试策略 | 默认无数次重试 |

## 5. 业务规则与验证逻辑

### 5.1 库区分配规则

| 规则ID | 规则描述 | 适用条件 |
|--------|---------|---------|
| ALLOC-01 | 有空闲杆子 → 按分拣维度(SKU/分色分码)分配 | 库区有空杆 |
| ALLOC-02 | 无空闲杆子 → 混色混码，进最少的杆子 | 库区无空杆 |
| ALLOC-03 | 一级库区为混合存储（多SKU） | 正常衣服 |
| ALLOC-04 | 二级库区为SKU精准存储 | 让道或精准需求 |
| ALLOC-05 | 缓存杆锁定单一SKU | 缓存策略 |

### 5.2 分流阀路由规则

| 规则ID | 规则描述 | 判断条件 |
|--------|---------|---------|
| ROUTE-01 | 插单款 → 跳过缓存杆，走快速通道 | 衣服标记为插单款 |
| ROUTE-02 | 需干燥 → 去干燥房 | 系统配置/方案判定 |
| ROUTE-03 | 正常衣服 → 进入分配流程 | 无特殊标记 |
| ROUTE-04 | 干燥房满 → 降级一级 | 干燥房无空位 |
| ROUTE-05 | 空衣架 → 回收到空衣架下架口 | 衣架上无衣服 |

### 5.3 下架呼叫优先级

```
优先级排序: ①转圈 > ②二级库区 > ③一级库区
```

| 优先级 | 来源 | 说明 |
|--------|------|------|
| ①最高 | 转圈（回源站） | 之前在系统中循环等待的衣服 |
| ②中等 | 二级库区 | 精准存储的SKU衣服 |
| ③最低 | 一级库区 | 混合存储的衣服 |

### 5.4 绑定方式差异

| 客户 | 上架绑定方式 | 说明 |
|------|------------|------|
| 荆门 | 自动绑定 | 系统自动绑定衣架信息 |
| 凯奥尼 | 自动绑定 | 系统自动绑定衣架信息 |
| 新基 | 手动绑定 | 操作员手动绑定衣架信息 |
| 安吉纳 | 手动绑定 | 操作员手动绑定衣架信息 |

### 5.5 验证规则汇总

| 规则ID | 验证规则 | 错误处理 |
|--------|---------|---------|
| VAL-01 | 未绑定加工方案 → 异常站 | 送入异常站下架口 |
| VAL-02 | 加工方案已下线 → 异常站 | 送入异常站下架口 |
| VAL-03 | 未绑定下架任务 → 异常站 | 送入异常站下架口 |
| VAL-04 | 目标满站 → 回源站转圈 | 重新入站等待 |
| VAL-05 | 1个存储杆同时只1个任务 | 冲突时排队 |
| VAL-06 | 1个分拣区域同时只1个存储杆执行 | 冲突时排队 |
| VAL-07 | 上架必须扫描衣架条码并确认入库 | 未扫描阻止入库 |
| VAL-08 | 下架必须扫描衣架条码并确认出库 | 未扫描阻止出库 |
| VAL-09 | 缺绑定方案/方案下线 → 快速通道→异常站 | 送入异常站 |

### 5.6 衣架条码扫描确认

- **上架确认**: 库区入口【扫描衣架条码→确认入库】
- **下架确认**: 下架口【扫描衣架条码→确认出库】
- 这两处是系统的人机交互确认点

## 10. 异常处理场景

### 10.1 上架异常

| 场景 | 可能原因 | 处理方式 |
|------|---------|---------|
| 短时间批量上架 | 系统压力 | 需要测试系统容量上限 |
| 上架口容量满 | 轨道拥堵 | 阻止继续上架，排队等待 |
| 上架扫描失败 | 条码损坏/RFID读取失败 | 人工干预 |

### 10.2 分流异常

| 场景 | 触发条件 | 处理方式 |
|------|---------|---------|
| 无法确定路由目标 | 缺绑定方案/方案下线 | 送入异常站下架口 |
| 目标满站 | 目标库区无空位 | 回源站转圈 |

### 10.3 库区分配异常

| 场景 | 触发条件 | 处理方式 |
|------|---------|---------|
| 一级库区无空杆 | 全部杆占用 | 混色混码进最少杆 |
| 二级库区无空杆 | 让道失败 | 回流到一级库区 |
| 缓存杆已锁定不同SKU | SKU不匹配 | 路由到其他可用杆 |

### 10.4 下架呼叫异常

| 场景 | 可能原因 | 处理方式 |
|------|---------|---------|
| 子任务超时 | 超过10分钟未完成 | 重试（默认无限次） |
| 无可用衣架 | 库存不足 | 等待新衣架上架 |
| 未绑定下架任务 | 系统配置缺失 | 送入异常站 |

### 10.5 设备异常

| 场景 | 可能原因 | 处理方式 |
|------|---------|---------|
| 电动线/分拣线故障 | 硬件故障 | 停机，人工处理 |
| RFID读取器故障 | 设备故障 | 手动扫码替代 |
| 工业平板离线 | 网络问题 | 恢复连接后重试 |

## 14. 安吉纳 (安吉纳) 分拣系统差异

> 来源: `安吉纳分拣系统用户手册.docx`
> 对比基线: 主线上方第1-13节

### 14.1 系统架构差异

安吉纳分拣系统采用 **PC管理配置 + PAD现场操作** 双层架构，与主线的硬件驱动视角有本质不同。

| 层面 | 主线 | 安吉纳 |
|------|------|--------|
| PC配置 | 分拣MES配置, 分拣下位配置 | + **分拣系统基础配置**（上架区/库区/下架区配置） |
| PAD菜单 | （未明确定义） | **后整 > 分拣管理, 上架位, 下架位** |
| 调试工具 | 无 | **调试 > 分拣线**（物流记录/站位调度/异常日志/操作日志） |
| 用户角色 | 上架/下架/包装站/异常处理/系统配置员 | **组长**(分拣管理+产线控制+站位管理) + **操作工**(上架/下架) |

### 14.2 菜单结构对比

**主线菜单**:
```
吊挂/
  ├── 流水线配置
  ├── 吊挂扫码终端配置
分拣/
  ├── 分拣MES配置
  └── 分拣下位配置
```

**安吉纳菜单**:
```
PC端:
  分拣MES配置      — 定义分拣线和站位信息，配置衣架容量、限制容量
  分拣下位配置      — 对分拣线设备信息进行维护管理
  分拣系统基础配置   — 对分拣系统上架区、库区、下架区进行配置 [安吉纳特有]

PAD端（后整菜单）:
  分拣管理/
    ├── 分拣任务    — 组长创建分拣任务，编排分拣工序，开启分拣流程
    ├── 产线控制    — 产线的开启、关闭（单线开关+批量全部开启/关闭）
    ├── 站位管理    — 查看站位数据、清空站内衣架容量、站位启用/禁用
    ├── 衣架路径    — 查看衣架路径信息、衣架事件、衣架更换
    └── 包装位详情  — 查看每站位下架任务情况 [安吉纳特有]
  上架位            — 衣架上架入口，操作工操作上架绑定
  下架位            — 衣架下架出口，组长呼叫下架

调试菜单:
  分拣线/
    ├── 物流记录    — 追踪查询衣架流转过程 [安吉纳特有]
    ├── 站位调度    — 点对点衣架调度 [安吉纳特有]
    ├── 衣架异常日志 — 异常衣架日志 [安吉纳特有]
    └── 用户操作日志 — 用户操作审计 [安吉纳特有]
```

### 14.3 任务生命周期差异（关键差异）

**主线模式**: 包装站下发装箱清单 → 系统生成父任务 → 拆分为子任务 → 子任务超时10min/无限重试

**安吉纳模式**: 组长从缝制任务单创建分拣任务，独立状态机:

```
新增任务(勾选缝制任务单) → 产前准备
  → [编辑: 分配上架区/下架区对应站位] → [上线] (保存配置并生效)
  → [发布] → 待分拣
  → [开始分拣] → 分拣中
  → (下架执行完成后任务结束)
```

关键操作语义:
- **保存**: 仅保存配置，不立即生效
- **上线**: 保存配置并正式生效
- **发布**: 任务状态变为"待分拣"
- **开始分拣**: 任务状态变为"分拣中"，可进行分拣

**对比总结**: 安吉纳没有父任务/子任务拆分概念，没有10min超时机制，没有无限重试逻辑。

#### 上架流程对比

| 步骤 | 主线 | 安吉纳 |

| 异常处理 | 异常站, 回源站转圈 | **取消**或**手动完成** |
| 辅助功能 | - | 颜色/尺码查询, 可呼叫数开关, 自动刷新时间设置 |

| 让道(不同款) | 二级满→回流到一级 | **不同款回流**（目标未明确） |


支持的调度类型:
- 上架位 → 库区
- 上架位 → 下架位
- 库区 → 库区
- 库区 → 下架位

### 14.8 安吉纳未实现的主线功能

| 主线功能 | 在安吉纳的状态 | 说明 |
|---------|--------------|------|
| 干燥房路径（场景二） | **不存在** | 无干燥房硬件，无静置计时 |
| 缓存杆路径（场景三） | **不存在** | 无缓存杆概念，无SKU锁定 |
| 插单款快速通道（场景四） | **不存在** | 无插单款标识，无快速通道 |
| 空衣架回流策略（场景五） | **未涉及** | 手册未提及空衣架如何回流 |
| 异常站路由（场景七） | **不存在** | 无异常站概念（但有衣架异常日志） |
| 父任务/子任务拆分 | **不存在** | 下架呼叫直接执行，不拆分 |
| 子任务10min超时+无限重试 | **不存在** | 无超时机制 |
| 存储杆并发约束(1杆1任务) | **未涉及** | 手册未提及 |
| 分拣区域并发约束(1区域1杆) | **未涉及** | 手册未提及 |
| 加工方案绑定检测 | **未涉及** | 手册未提及加工方案概念 |
| 衣架条码上架确认入库 | **部分不同** | 上架时有扫码绑定，但无"库区入口扫描确认"步骤 |
| 衣架条码下架确认出库 | **不存在** | 无下架口的扫描确认环节 |

### 14.10 测试关注点（安吉纳特定）

1. **分拣任务状态机**: 验证 产前准备→待分拣→分拣中 的完整流转，特别关注保存vs上线的区别
2. **产线控制**: 验证单线开关和批量全部开启/关闭
3. **站位管理**: 验证站位启用/禁用后的操作阻断，衣架清除和校准功能
4. **衣架更换**: 验证损坏衣架更换后新衣架是否正确继承原分拣路径
5. **站位调度**: 验证4种调度类型的正确性（上架位→库区/下架位, 库区→库区/下架位）
6. **下架位**: 验证可呼叫数过滤开关、自动刷新、取消/手动完成的异常处理
7. **一级库区路由**: 验证同色同款存放、同款不足混款、让道同款→二级、让道不同款→回流
8. **二级库区优先级**: 验证包装下架触发优先选择二级库区
9. **上架位**: 验证物理提升按钮+扫码绑定的完整流程，日产量和已上架数计数
10. **衣架路径**: 验证分拣路径查询、缝制路径关联、衣架事件查看
