# 瑞晟IMS ERP 页面数据捕捉操作指南

> 用途：供拥有系统访问权限的人员，对5个盲区模块进行完整的页面数据捕捉。
> 目标：将捕捉到的数据追加到 `module_texts.txt`，填补知识盲区。

---

## 1. 捕捉输出格式规范

输出文件必须严格遵循 `module_texts.txt` 的现有格式。

### 1.1 模块头部

```
===== NN_模块名 (XXXXX chars) =====
```

- `NN` = 两位数字模块序号，尾随下划线。盲区模块推荐序号：07_物流、11_样衣、12_平板、13_人事、14_看板
- `模块名` = 中文模块名称
- `XXXXX chars` = 该模块节的实际字符总数（含换行符），可用文本编辑器统计

### 1.2 全局头部（每个模块节重复出现）

```
瑞晟智能制造协同平台
最佳智造
99+
```

- `99+` 是通知徽章计数，捕捉时记录实际数值即可

### 1.3 侧边栏模块列表

按实际出现顺序每行一个，空行分隔：

```
gm.样衣#1

gm.销售#1

gm.采购#2

...（全部14个模块）
```

### 1.4 菜单区域

- 二级菜单分组名：顶格书写，不缩进
- 三级页面名：每行一个，前加两个空格缩进
- 分组之间用空行分隔

```
订单
  销售订单
  销售订单

排期
  订单预排期
  订单排期表
```

- **重复项必须保留** — 原始数据中存在重复页面名，不要去重

### 1.5 分隔符与全局通知区域

```
/
gm.首页#2
ML863
生产订单
ML870
签证到期提醒
  [员工编号，姓名，签证到期日期...]
2026/06/30 08:00:13
护照到期提醒
  [员工编号，姓名，护照到期日期...]
```

- `/` 独占一行，标记模块菜单结束
- 之后为全局通知区域，内容随登录时间和角色变化
- 记录时保留完整原始文本

## 2. 每个页面的详细捕捉清单

对每个可访问的页面，按以下清单逐项记录。**页面与页面之间用 `---` 分隔线隔开。**

### 2.1 页面元信息

```
### [页面名称]
- URL: [浏览器地址栏完整路径，含查询参数]
- 标题: [浏览器标签页标题]
- 面包屑: 首页 > [模块名] > [菜单分组] > [当前页面]
```

### 2.2 表单字段（如页面含表单）

每个字段一行，格式：

```
| 序号 | 字段标签 | 字段类型 | 必填 | 默认值 | Placeholder | 备注 |
|------|---------|---------|------|--------|-------------|------|
| 1 | 订单编号 | text input | 是 | 自动生成 | 请输入 | 只读灰色 |
```

字段类型枚举：`text input`, `textarea`, `select/dropdown`, `date picker`, `datetime picker`, `number input`, `checkbox`, `radio`, `file upload`, `switch/toggle`, `read-only display`

### 2.3 下拉框/选择框的完整选项

```
下拉框: [字段标签]
  选项列表（逐行记录）:
  - [选项值1]
  - [选项值2]
  - ...
```

### 2.4 表格列定义

```
表格: [表格名称或位置描述]
| 序号 | 列标题 | 可排序 | 可筛选 | 列宽(估) | 备注 |
|------|--------|--------|--------|----------|------|
| 1 | 序号 | 否 | 否 | 60px | 行号列 |
| 2 | 订单编号 | 是 | 是 | 150px | 可点击跳转 |
```

### 2.5 按钮清单

```
按钮:
  [按钮文本] - 位置: [表格上方/表单底部/行内操作列] - 类型: [主按钮/次按钮/危险按钮/链接按钮] - 可见条件: [始终可见/选中行后出现/特定状态可见]
```

### 2.6 状态标签与颜色

```
状态标签:
  [状态文本] - 颜色: [红色#FF0000 / 绿色#00FF00 / 橙色 / 蓝色 / 灰色] - 出现位置: [列表页状态列/详情页头部]
```

### 2.7 筛选/搜索区域

```
筛选条件:
  - [字段名]: [类型] - [placeholder或选项范围]
  - ...
搜索栏位置: [表格上方/页面顶部]
```

### 2.8 页签/Tab

```
Tab列表（如页面含Tab切换）:
  - [Tab1名称] (默认激活)
  - [Tab2名称]
  - ...
```

### 2.9 提示/校验/工具提示

```
提示信息:
  - 字段: [字段名] - 悬浮提示: [tooltip文本]
  - 字段: [字段名] - 校验规则: [如"必填"触发时的红色提示文本]
  - 操作: [按钮名] - 确认弹窗文本: [弹窗内容]
```

### 2.10 截图引用

```
截图:
  - 全页截图: [模块名]_[页面名]_full.png
  - 表单区域: [模块名]_[页面名]_form.png
  - 表格区域: [模块名]_[页面名]_table.png
  - 下拉选项: [模块名]_[页面名]_dropdown_[字段名].png
  - 状态标签: [模块名]_[页面名]_status.png
```

### 3.3 平板/PAD模块 (gm.平板#1) — P2

**当前已知：** 模块名称存在，覆盖率约15%

| 优先级 | 页面/区域 | 说明 |
|--------|----------|------|
| **必须** | PC端访问时显示什么 | 可能显示"请在PAD设备上访问"或显示有限功能 |

| **建议** | PAD端实际访问时的页面差异 | 移动端UI通常有简化表单、大按钮、扫码功能 |
| **建议** | 离线模式相关UI | PAD常用于车间现场，是否有离线缓存提示 |
| **建议** | 与生产/仓库模块的重叠功能 | PAD通常用于现场扫码、报工、入库确认等 |

**捕捉方式：**
1. 先PC端访问 `gm.平板#1`，记录PC端看到的内容
2. 如果条件允许，使用PAD设备或浏览器移动端模拟（F12 > 设备模拟 > iPad）访问
3. 对比两端差异，分别记录

## 4. 分步捕捉操作流程（10步）

### 步骤 1：准备工作

```
1. 打开 Chrome/Edge 浏览器，新建无痕窗口
2. 登录 IMS 系统（记录登录角色名: _______________）
3. F12 打开 DevTools，切换到以下标签页（各开一个）：
   - Elements（查看DOM结构）
   - Network（记录API请求，XHR/Fetch过滤）
   - Console（观察错误日志）
4. 在 DevTools > Network 中勾选 "Preserve log"（页面跳转时保留日志）
5. 新建文本文件作为当次捕捉输出文件
```

### 步骤 2：记录模块头部与全局框架

```
1. 点击目标模块侧边栏项（如 gm.物流#1）
2. 等待页面完全加载（Network标签无pending请求）
3. 记录到输出文件：
   - 模块头部: "===== 07_物流 (XXXXX chars) =====" （chars稍后填写）
   - 应用标题、徽章数
   - 完整的侧边栏模块列表（14项，每次模块切换都会重新出现）
4. 截图侧边栏完整展开状态，文件名: 00_sidebar_full.png
```

### 步骤 3：捕捉完整菜单树

```
1. 在目标模块内，展开所有二级菜单分组
2. 如有折叠/展开箭头，确保全部展开
3. 按顺序记录每个二级菜单分组名和三级页面名
4. 记录分隔符 "/" 之后的所有全局通知内容
5. 截图菜单完整展开状态，文件名: 00_menu_tree.png
```

### 步骤 4：逐页抓取 — 页面元信息

```
对每个三级页面：
1. 点击进入页面
2. 等待完全加载
3. 在输出文件中创建 "### [页面名]" 标题
4. 从地址栏复制完整URL，记录到 - URL: 行
5. 从标签页标题复制，记录到 - 标题: 行
6. 查看页面顶部面包屑导航，记录完整路径
7. 截图全页: [模块]_[页面]_full.png
```

### 步骤 5：逐页抓取 — 筛选/搜索区

```
1. 定位搜索/筛选区域（通常在表格上方或页面顶部）
2. 记录每个筛选字段的标签、类型、placeholder
3. 展开所有下拉筛选框，记录完整选项列表
4. 点击"更多筛选"/"高级搜索"等按钮，记录隐藏的筛选条件
5. 截图筛选区: [模块]_[页面]_filter.png
```

### 步骤 6：逐页抓取 — 表格列（列表页）

```
1. 滚动表格至最右，确保看到所有列
2. 逐列记录：列标题、是否有排序箭头(↕)、是否有筛选图标(漏斗)
3. 点击任意列的排序图标，观察排序方向和效果
4. 点击列筛选图标（如有），记录可用的筛选项
5. 记录操作列中的按钮（编辑/删除/查看/审批等）
6. 截图表格全宽: [模块]_[页面]_table.png
```

### 步骤 7：逐页抓取 — 表单字段（详情/新增/编辑页）

```
1. 点击"新增"或"编辑"按钮进入表单视图
2. 按页面顺序从上到下、从左到右记录每个字段
3. 对每个字段确认：
   - 标签旁是否有红色 *（必填标记）
   - 输入框内的灰色 placeholder 文本
   - 下拉框的所有选项（逐一点开记录）
   - 日期选择器的格式（如 YYYY-MM-DD）
   - 字段是否有灰色只读态
4. 鼠标悬浮在字段标签旁的 ? 图标上，记录 tooltip
5. 截图表单: [模块]_[页面]_form.png
```

### 步骤 8：逐页抓取 — 按钮、状态、页签

```
1. 记录页面上所有可见按钮及其位置
2. 条件按钮（如选中行后出现的按钮）：选中一行后记录新出现的按钮
3. 观察状态列的颜色标签，记录所有出现的状态值
4. 如有顶部Tab切换，依次点击每个Tab并：
   - 记录Tab名称
   - 记录Tab下的内容变化（新表格/新表单）
   - 截图每个Tab
5. 截图状态标签集合: [模块]_[页面]_status.png
```

### 步骤 9：捕捉API接口调用

```
1. 在 Network 标签页中，筛选 XHR/Fetch 请求
2. 进入每个页面时，注意触发的API调用
3. 记录关键接口路径（忽略token/timestamp参数）：
   - 列表查询: GET /api/xxx/list?page=1&size=20
   - 详情查询: GET /api/xxx/detail?id=xxx
   - 提交保存: POST /api/xxx/save
   - 状态变更: PUT /api/xxx/status
4. 可选：右键 > Copy > Copy as cURL，保存到单独文件
```

### 步骤 10：收尾与统计

```
1. 回顾所有页面，确认没有遗漏
2. 用文本编辑器统计输出文件的字符数（含换行符）
3. 将实际字符数填入模块头部 "===== NN_模块名 (XXXXX chars) ====="
4. 检查格式完整性：
   ✓ 模块头部字符数已填写
   ✓ 所有页面有 "### [页面名]" 标题
   ✓ 页面间有 "---" 分隔线
   ✓ 侧边栏14个模块均已列出
   ✓ "/" 分隔符已记录
   ✓ 全局通知内容已记录
   ✓ 所有截图文件已保存并命名
```

## 6. 捕捉会话检查清单

每次捕捉会话前打印/复制此清单，逐项勾选：

```
=============================
  页面捕捉检查清单
  日期: _______________
  模块: _______________
  操作人: _______________
  登录角色: _______________
=============================

【准备】
□ 无痕浏览器窗口已打开
□ DevTools 已打开 (Elements / Network / Console)
□ Network > Preserve log 已勾选
□ 输出文本文件已创建并写入模块头部
```

## 附录B：常见字段类型判断指南

| 页面表现 | 字段类型 | 记录要点 |
|---------|---------|---------|
| 白色输入框 + 光标可进入 | text input | 记录 placeholder、maxlength |
| 多行白色输入框 | textarea | 记录行数（rows）、maxlength |
| 灰色背景输入框 + 不可编辑 | read-only display | 标注"只读" |
| 右侧有向下箭头 + 点击弹出选项列表 | select/dropdown | 逐项列出所有选项 |
| 点击弹出日历面板 | date picker | 记录日期格式 |
| 右侧有向上/向下箭头 | number input | 记录最小/最大值、步长 |
| 方形勾选框 | checkbox | 记录勾选/不勾选的含义 |
| 圆形单选按钮 | radio | 记录所有互斥选项 |
| "上传"或云图标按钮 | file upload | 记录允许的文件类型 |
| 滑动开关 | switch/toggle | 记录ON/OFF对应的含义 |
