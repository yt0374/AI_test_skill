## 3. 床次 (Bed Number) Data Profiles

| Profile | 拉布层数 | 相对于剩余层数 | 预期结果 |
|---------|----------|---------------|----------|
| 正常 | 20 | 剩余=50, 20<50 | 通过 |
| 刚好 | 50 | 剩余=50, 50=50 | 通过，剩余=0 |
| 预警边界 | 41 | 剩余=50, 41<50但剩余9<10 | 通过但触发预警 |
| 刚好预警 | 40 | 剩余=50, 40<50但剩余=10 | 通过，不预警（边界值10本身） |
| 超裁 | 51 | 剩余=50, 51>50 | 强控禁止 |
| 剩余为负 | 60 | 剩余=50, 60>50 | 强控，"总裁剪层数已超计划" |
| 零层数 | 0 | 剩余=50 | 应拒绝（无效输入） |
| 负数层数 | -5 | 剩余=50 | 应拒绝（无效输入） |
| 非整数 | 20.5 | 剩余=50 | 应拒绝（需求未明确，但推测需正整数） |

## 7. 状态枚举

| 实体 | 状态值 | 说明 |
|------|--------|------|
| 裁剪任务 | 未开始 / 拉布中 / 已完成 | PC端创建后PAD端同步 |
| 床次 | 拉布中 / 已拉布 | 暂扣层数 vs 锁定层数 |
| 扎卡类型 | 通用扎卡 / 指定扎卡 | 取决于CPO合并状态 |
| 充数模式 | 自动 / 手动 | PAD端可切换 |
