> 客户：荆门鹰美 (jingmen)
> 文档类型：UI-heavy（页面流程 + 表单 + 状态机）

## 1. Overview + Business Context

### 1.1 改造背景

荆门鹰美原有裁剪系统采用 CPO（Customer Purchase Order）强制分配策略——每个扎卡必须绑定唯一 CPO，缝制上线严格按 CPO 组织。在实际生产中暴露出两个核心问题：

- **交期冲突**：CPO 交期临时提前时，原系统难以快速响应
- **现场适应性差**：内销单、多色号混合裁剪等场景不需要严格区分 CPO

### 1.2 改造目标

从"强制分配到 CPO"转变为"按需绑定 CPO"——需要精确追踪的订单保留 CPO 控制，不需要的场景可自由合并。

## 2. Functional Points

### F001 - 裁剪任务新增 CPO 维度与合并
用户创建裁剪任务时可选择一个或多个 CPO，或按交期智能合并。系统基于所选 CPO 的订单数量生成裁剪任务。

### F002 - 唛架计划层数分配与递减
一个唛架可被多个裁剪任务引用。分配给所有任务的计划层数之和不得超过总计划层数。系统自动递减可用层数并防超分配。

### F003 - 顶层裁剪任务管理（PC端）
PC端创建裁剪任务，关联 CPO、计划层数、计划件数、生产组别。发布后 PAD 端同步可见。

### F004 - 床次创建与提交（PAD端）
拉布员在 PAD 端进入裁剪任务后点击【新增床次】，扫描布卷号，输入拉布层数。系统实时校验防超裁。

### F005 - 防超裁机制（预警 + 强控 + PC审核）
剩余层数低于10层时预警；输入导致剩余为负时禁止提交；PC端可申请超裁审核。

### F006 - 床次号生成与管理
床次号在【提交完成】时自动生成，生产订单内全局唯一。格式：主布 C 开头，辅布 F 开头。

### F007 - 扎卡打印与 CPO 关联
合并 CPO 场景隐藏 CPO 信息（通用扎卡）；单一 CPO 场景强制显示 CPO 号（指定扎卡）。

### F008 - 挂片站自动充数模式
系统根据 CPO 交期顺序自动充数，优先分配交期最紧急的 CPO，满额后自动切换。

### F009 - 挂片站手动充数模式
员工可在 PAD 端切换至手动模式，自由选择未满额 CPO 进行挂片，CPO 满额时提示切换。

### F010 - 自动/手动模式切换与联动
员工随时可在自动和手动模式间切换，系统保存进度并提示。

## 3. Business Rules

### R001 - CPO 合并规则
- 勾选方式：可勾选一个或多个 CPO
- 全选：一键选中当前订单所有 CPO
- 智能合并：系统根据交期自动将同交期 CPO 归为一组

### R002 - 唛架层数分配规则
- 唛架作为公共资源，可被多个裁剪任务引用
- 分配给所有裁剪任务的计划层数之和 ≤ 总计划层数
- 超出部分禁止提交

### R003 - 层数递减公式
```
唛架可用层数 = 总计划层数 - Σ(已分配给各裁剪任务的层数)
裁剪任务剩余可用层数 = 计划层数 - Σ(已拉布锁定层数) - Σ(拉布中暂扣层数)
```

### R004 - 防超裁规则
- 软预警阈值：剩余层数 < 10 层 → "剩余层数不足X层，请注意控制。"
- 硬拦截：输入层数导致剩余为负 → "总裁剪层数已超计划，需PC端审核。"
- PC审核：放行（增加计划层数）或驳回（调整层数）

### R005 - 床次编辑控制
- 未开始：可编辑或删除
- 拉布中（任一床次）：编辑/删除按钮隐藏

### R006 - 床次号规则
- 作用域：生产订单内全局唯一
- 生成时机：【提交完成】时自动生成
- 主布 C 开头，辅布 F 开头，格式：床次-001
- 同一秒多次提交以数据库记录ID严格递增

### R007 - 扎卡 CPO 显示规则
- 合并 CPO：隐藏 CPO 选择区域，扎卡不印 CPO 号 → 通用扎卡
- 单一 CPO：强制显示 CPO 号，扎卡印刷 CPO 号 → 指定扎卡

### R008 - 挂片自动充数规则
- 通用扎卡：按交期顺序自动充入最紧急 CPO，满额后切换
- 指定扎卡：第一阶段优先绑定指定 CPO；满额后第二阶段按交期顺延

### R009 - 挂片手动充数规则
- 手动模式下可跳过交期顺序自由选择
- CPO 满额时提示"CPO [编号] 数量已满，请切换其他 CPO。"
- 不切换则禁止继续挂片到满额 CPO

### R010 - 分扎计件规则
- 合并 CPO：按布卷号分扎，扎卡记录布卷号和层数
- 单一 CPO：按 CPO 件数分扎，每个 CPO 独立一扎

### R011 - 提交完成校验
- 实际层数 ≤ 原暂扣层数 + 当前实时剩余可用层数 → 通过
- 否则失败：暂扣保持，提示"实际层数超出可调整范围"

## 4. State Machines

### 4.1 裁剪任务状态机

```
┌──────────┐     有床次创建      ┌──────────┐    所有床次已完成     ┌──────────┐
│  未开始   │ ─────────────────→ │  拉布中   │ ─────────────────→ │  已完成   │
│          │                    │          │  且剩余层数=0       │          │
└──────────┘                    └──────────┘                    └──────────┘
     │                               │
     │ 可编辑/删除                    │ 编辑/删除隐藏
     │                               │
```

**状态定义：**
| 状态 | 条件 | 行为 |
|------|------|------|
| 未开始 | PC端创建，无床次 | 可编辑、可删除 |
| 拉布中 | 至少有一个床次为"拉布中"或"已拉布" | 编辑/删除按钮隐藏 |
| 已完成 | 所有床次已提交，剩余层数=0 | 只读 |

### 4.2 床次状态机

```
                【新增床次】                  【提交完成】
┌──────────┐                    ┌──────────┐                    ┌──────────┐
│  (创建)   │ ─────────────────→ │  拉布中   │ ─────────────────→ │  已拉布   │
│          │   暂扣层数          │          │   锁定层数          │          │
└──────────┘                    └──────────┘                    └──────────┘
```

**状态定义：**
| 状态 | 触发条件 | 层数影响 |
|------|----------|----------|
| 拉布中 | 点击【新增床次】成功 | 暂扣层数（参与剩余计算） |
| 已拉布 | 点击【提交完成】成功 | 锁定层数（永久占用） |

### 4.3 防超裁状态机

```
                   剩余层数 < 10                    剩余层数 < 0
┌──────────┐                         ┌──────────┐                         ┌──────────┐
│  正常     │ ─────────────────────→  │  预警     │ ─────────────────────→  │  强控     │
│          │                         │ (软提醒)  │                         │ (禁止提交) │
└──────────┘                         └──────────┘                         └─────┬────┘
                                                                            │
                                                                    ┌───────┴───────┐
                                                                    │  PC端超裁审核  │
                                                                    │ 放行 / 驳回    │
                                                                    └───────────────┘
```

## 7. Edge Cases & Constraints

### 唛架层数
- 唛架总层数=0：不可创建任何裁剪任务
- 分配层数=总层数+1：超分配，禁止提交
- 分配层数=总层数（刚好）：允许，剩余=0
- 多任务并发分配同一唛架：需防竞态

### 床次创建
- 布卷号为空：拒绝创建
- 拉布层数=0：应拒绝（无意义）
- 拉布层数=负数：应拒绝
- 拉布层数=剩余值刚好：允许
- 拉布层数=剩余值+1：拒绝（强控）
- 并发创建床次：剩余层数计算需防竞态

### 床次号
- 生产订单内第一个床次：床次-001 或 C-001
- 同一秒并发提交：以数据库记录ID确保严格递增
- 最大床次号溢出：需定义上限（如 9999）

### 挂片充数
- CPO 数量=0：应跳过该 CPO
- 所有 CPO 满额：提示无可挂片 CPO
- 手动模式下切换 CPO 时系统中断：进度需保存
- 自动模式下交期相同的 CPO：需定义排序次优先级

### 状态边缘
- 裁剪任务状态判断使用床次聚合条件时，所有床次并发状态变更瞬间
- 已有"已拉布"床次时尝试删除裁剪任务（应隐藏按钮）
