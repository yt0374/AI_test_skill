## 1. 概述与业务背景

本文档定义了IMS v3.x对外（ERP）的标准接口协议，用于千虹客户ERP与IMS仓库管理系统之间的数据交互。核心场景为面料/辅料/成品/裁片的仓库全生命周期管理。

**协作模式**: ERP作为上游业务系统，IMS作为智能仓库执行系统。采用API（请求下发）+ SPI（异步回调）双通道通信。

**物料流转规范**（严格单向）:
```
[4.4 物料清单同步] → [4.6 入库] → [已入库] → [4.5 载具绑定] → [4.6 出库] → [已出库]
```

## 2. 功能点列表

### 2.1 认证模块

| ID | 功能点 | 说明 |
|----|--------|------|
| **F001** | 获取访问令牌 | POST /public/auth/authorization，username+password获取Bearer Token |
| **F002** | Token携带鉴权 | 所有业务API请求头携带 Authorization: Bearer {token} |

### 2.2 WMS载具模块

| ID | 功能点 | 说明 |
|----|--------|------|
| **F003** | 载具信息查询 | POST /wms/v3/carrier/info，按carrierNo查询，支持3级深度(基本信息/任务/物料) |
| **F004** | 载具任务下发 | POST /wms/v3/carrier/task/execute，支持上架/下架/库外移动三种任务类型 |
| **F005** | 载具任务结果回调(SPI) | IMS→ERP异步回调，通知任务执行完成(FINISHED) |

### 2.3 WMS库区模块

| ID | 功能点 | 说明 |
|----|--------|------|
| **F006** | 库区查询(模糊) | POST /wms/v3/warehouse/space，按库区编码模糊查询+属性精准匹配 |
| **F007** | 库区查询(按仓库类型) | POST /wms/v3/warehouse/space/list，按仓库类型编码查库区列表 |

### 2.4 WMS物料模块

| ID | 功能点 | 说明 |
|----|--------|------|
| **F008** | 物料清单同步 | POST /wms/v3/material/item/sync，同步物料物品清单(含initInventory选项) |
| **F009** | 物料清单同步结果回调(SPI) | IMS→ERP异步回调，返回syncState+successList+failureList |
| **F010** | 货载绑定/解绑 | POST /wms/v3/carrier/item/link，4种操作组合(bindType x linkMode) |
| **F011** | 物料出入库(通用) | POST /wms/v3/material/item/stock/flow，flowType区分入库(1)/出库(2) |
| **F012** | 物料入库(独立) | POST /wms/v3/material/item/inbound，单一入库指令 |
| **F013** | 物料出库(独立) | POST /wms/v3/material/item/outbound，单一出库指令 |
| **F014** | 物料库存查询 | POST /wms/v3/material/item/inventory，分页查询库存(含扩展属性) |

### 2.5 生产模块

| ID | 功能点 | 说明 |
|----|--------|------|
| **F015** | 生产订单同步 | POST /production/v3/save，同步生产订单(含色码明细+工序列表) |
| **F016** | 缝制任务单同步 | POST /sewing/v3/save，同步缝制任务单(含色码明细) |

### 2.6 视图模块

| ID | 功能点 | 说明 |
|----|--------|------|
| **F017** | 工序产量视图 | 数据库视图，记录每站每衣架的产出数据 |
| **F018** | 质检详情视图 | 数据库视图，记录每站质检数据 |

## 3. 业务规则

| ID | 规则 | 来源 |
|----|------|------|
| **R001** | 物料必须先入库(flowType=1)才能出库(flowType=2) | 4.6节 |
| **R002** | 入库时可同时更新itemList中物料的字段，IMS以入库请求中的值为准覆盖 | 4.6节 |
| **R003** | 物料必须处于"已入库"状态才能绑定到载具 | 4.5节 |
| **R004** | 载具绑定BIND+FULL = 先解绑全部已有物料，再绑定新物料 | 4.5节 |
| **R005** | 载具绑定BIND+INCR = 保留已有绑定物料，追加新物料 | 4.5节 |
| **R006** | 载具解绑UNBIND+INCR = 按itemList逐项解绑指定物料 | 4.5节 |
| **R007** | 载具解绑UNBIND+FULL = 解绑载具上所有物料（不传itemList） | 4.5节 |
| **R008** | 载具任务执行后异步回调结果，回调失败重试最多3次(指数退避) | 5.0节 |
| **R009** | SPI回调ERP须在5秒内返回HTTP 200 + code=0 | 5.0节 |
| **R010** | 所有业务API须携带 Bearer Token（除获取Token接口本身） | 3.2节 |
| **R011** | 每个请求须携带 Metas-Request-Id（UUID）作为唯一标识 | 3.3节 |
| **R012** | API同步返回code=0仅表示受理成功，非业务处理成功 | 3.5节 |
| **R013** | Token有效期及刷新机制另行约定 | 3.2节 |

## 6. 数据实体与变化维度

### 6.1 载具

| 维度 | 取值 | 测试场景 |
|------|------|----------|
| 载具类型 | 01面料托盘, 02松布架, 03裁片箱, 04成品箱, 07辅料箱, 08料架 | 每种类型的行为验证 |
| 载具状态 | 0初始, 1上架中, 2已上架, 3下架中, 4已下架, 5移库中, -1异常 | 状态转换合法性 |
| 绑定状态 | 0空, 1非空 | 绑定/解绑前置条件 |
| 查询深度 | 1基本信息, 2+任务, 3+物料 | 不同深度的响应差异 |

### 6.2 物料

| 维度 | 取值 | 测试场景 |
|------|------|----------|
| 物料类型 | fabric, trim, garment, cuttingPieces | 不同物料的扩展属性差异 |
| 出入库类型 | 1入库, 2出库 | 正向流程+反向约束 |
| 初始化库存 | true/false | 同步后是否自动入库 |
| 单位 | M, KG, PCS | 数量计算校验 |

### 6.3 任务

| 维度 | 取值 | 测试场景 |
|------|------|----------|
| 任务类型 | PUT_ON, PUT_DOWN, MOVE_OUTSIDE | 每种任务下发 |
| 任务状态 | WAIT, WORKING, CANCEL, FINISH, ERROR | 状态转换+SPI回调 |
| 绑定模式 | BIND/UNBIND x FULL/INCR | 4种组合操作 |

## 7. 边界条件与约束

| ID | 边界/约束 | 说明 |
|----|----------|------|
| **E001** | 未入库物料不可出库 | 调用flowType=2时itemNo未入库 |
| **E002** | 未入库物料不可绑定 | 调用bind时itemNo未入库 |
| **E003** | Token过期后API拒绝 | 过期Token调用业务API |
| **E004** | 不存在的carrierNo查询 | 返回空还是错误？ |
| **E005** | 不存在的itemNo出入库 | 受理成功但后续失败？ |
| **E006** | 已绑定的载具再次FULL绑定 | 先清空再绑定新物料 |
| **E007** | 重复requestId的幂等性 | 同一requestId重复提交 |
| **E008** | pageSize超过100的库存查询 | 是否截断到100？ |
| **E009** | pageNo为0或负数的库存查询 | 默认1还是报错？ |
| **E010** | 空itemList的同步请求 | 是否允许空数组？ |
| **E011** | SPI回调ERP超时(>5s) | IMS重试+最终失败处理 |
| **E012** | 非法taskType的任务下发 | 不在{PUT_ON,PUT_DOWN,MOVE_OUTSIDE}中 |
| **E013** | 非法bindType/linkMode组合 | FULL+UNBIND+itemList非空 |
