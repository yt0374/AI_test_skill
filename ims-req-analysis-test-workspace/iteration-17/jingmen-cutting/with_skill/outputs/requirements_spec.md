# Requirements Specification: 荆门新裁剪2.0

## 1. Overview + Business Context

### 1.1 项目背景

荆门鹰美工厂原有裁剪系统基于 CPO（Customer Purchase Order）强制分配策略，每个扎卡必须打印 CPO 号，缝制上线时必须严格按 CPO 组织生产。该模式在固定大订单、单一交期场景下可运行，但在实际生产中暴露以下痛点：

- **交期冲突**：CPO 交期临时提前时，原系统难以快速响应。
- **现场适应性差**：内销单、多色号混合裁剪等场景不需要严格区分 CPO，强制绑定反而增加复杂度。

### 1.2 改造目标

从"强制分配到 CPO"转变为"按需绑定 CPO"，实现松紧有度——需要精确追踪的订单保留 CPO 控制，不需要的场景可自由合并。

## 2. Functional Points

### F001 - 裁剪任务创建（CPO 选择与合并）
裁剪任务创建时新增 CPO 选择框，支持：
- 勾选单个或多个 CPO
- 全选当前订单所有 CPO
- 智能合并：系统根据交期将同交期 CPO 自动归组

### F002 - 唛架计划层数分配与递减
- 一个唛架可被多个裁剪任务引用
- 用户为 CPO（或合并组）创建裁剪任务时，从唛架总计划层数中划拨层数
- 已分配后剩余可用层数自动递减
- 系统强制校验：所有任务的计划层数之和不得超过唛架总计划层数

### F003 - 顶层裁剪任务管理（PC端）
- PC 端创建裁剪任务作为"顶层拉布任务"
- 任务关联 CPO、计划层数、计划件数、生产组别
- 任务状态：未开始、拉布中、已完成
- 未开始时可编辑或删除；拉布中后编辑/删除按钮隐藏

### F004 - 现场床次创建（PAD端）
- 拉布员在 PAD 端进入裁剪任务后点击【新增床次】
- 录入布卷号（扫描+手动备用）和拉布层数
- 系统实时计算剩余可用层数并校验
- 床次创建成功后状态为"拉布中"，暂扣层数

### F005 - 床次提交完成（PAD端）
- 拉布完成后录入实际拉布层数
- 系统锁定实际层数，床次状态更新为"已拉布"
- 校验实际层数不超过可调整范围
- 床次号在提交完成时正式生成

### F006 - 床次号唯一性管理
- 床次号在一个生产订单号内全局唯一
- 格式：主布 C 开头，辅布 F 开头（如 床次-001）
- 生成时机：提交完成时，按数据库记录 ID 严格递增
- 同一生产订单下多个裁剪任务共享递增序列

### F007 - 防超裁机制
- 软预警：剩余层数低于预设阈值（如 10 层），PAD 弹出提示
- 强控禁止：层数导致剩余为负，禁止创建/提交，提示需 PC 端审核
- PC 端超裁审核：负责人可放行（增加计划层数）或驳回

### F008 - 扎卡打印（CPO 关联）
- 场景 A（合并 CPO）：隐藏 CPO 信息，不印刷 CPO 号 → 通用扎卡
- 场景 B（单一 CPO）：强制显示 CPO 号，扎卡上印刷 CPO 号 → 指定扎卡
- 分扎规则：合并 CPO 按布卷号分扎；单一 CPO 按 CPO 件数分扎

### F009 - 缝制挂片自动充数模式
- 通用扎卡：系统按交期顺序自动充入最近 CPO，满额后自动切换
- 指定扎卡：优先充入指定 CPO，满额后顺延按交期切换
- 员工全程无需手动操作

### F010 - 缝制挂片手动充数模式
- 员工可自主切换至手动模式
- 展示所有未满额 CPO 列表（按交期排序）
- 自由选择任意未满额 CPO（无交期限制）
- CPO 满额时弹出提示并要求切换
- 支持随时切回自动模式

## 3. Business Rules

| ID | 规则 | 类型 | 模块 |
|----|------|------|------|
| R001 | 裁剪任务仅针对所选 CPO 的订单数量创建 | 核心逻辑 | 裁剪任务 |
| R002 | 唛架已分配层数 = SUM(所有裁剪任务分配的计划层数) | 计算公式 | 裁剪任务 |
| R003 | 唛架剩余可用层数 = 总计划层数 - 已分配层数 | 计算公式 | 裁剪任务 |
| R004 | 所有裁剪任务的计划层数之和 ≤ 唛架总计划层数（禁止超分配） | 强校验 | 裁剪任务 |
| R005 | 剩余可用层数 = 计划层数 - SUM(已拉布锁定层数) - SUM(拉布中暂扣层数) | 计算公式 | 床次任务 |
| R006 | 输入层数 ≤ 剩余可用层数 → 校验通过；否则 → 防超裁机制 | 实时校验 | 床次任务 |
| R007 | 提交完成时：实际层数 ≤（原暂扣层数 + 当前实时剩余可用层数）→ 通过 | 提交校验 | 床次任务 |
| R008 | 床次号在生产订单号内全局唯一，提交完成时生成，按 DB 记录 ID 递增 | 唯一性规则 | 床次任务 |
| R009 | 拉布中状态禁止编辑/删除裁剪任务 | 状态门控 | 裁剪任务 |
| R010 | 合并 CPO 场景：扎卡不印刷 CPO 号，挂片按交期自动分配 | 业务规则 | 扎卡打印 |
| R011 | 单一 CPO 场景：扎卡印刷 CPO 号，优先绑定指定 CPO，满额后顺延 | 业务规则 | 扎卡打印 |
| R012 | 自动充数：通用扎卡按交期排序自动分配；指定扎卡优先绑定后顺延 | 分配逻辑 | 缝制挂片 |
| R013 | 手动充数：员工自由选择未满额 CPO，满额禁止继续挂片 | 选择逻辑 | 缝制挂片 |
| R014 | 模式切换：自动→手动保存进度；手动→自动重新计算交期排序分配 | 联动逻辑 | 缝制挂片 |

## 4. Data Flow

```
PC端（管理）
  │
  ├── 生产订单 → [选择CPO(单个/合并)] → 创建裁剪任务
  │     ├── 关联唛架 → 划拨计划层数 → 唛架剩余层数递减
  │     └── 发布裁剪任务 → PAD同步可见
  │
  ├── 扎卡打印
  │     ├── 合并CPO → 隐藏CPO → 通用扎卡 → 按布卷号分扎
  │     └── 单一CPO → 显示CPO → 指定扎卡 → 按CPO件数分扎
  │
  └── 超裁审核（放行/驳回）
        └── 放行 → 增加计划层数

PAD端（现场）
  │
  ├── 裁剪任务列表 [未开始/拉布中/已完成]
  │     └── 进入裁剪任务 → 新增床次
  │           ├── 扫描布卷号 + 输入拉布层数
  │           ├── 实时校验：层数 ≤ 剩余可用层数
  │           │     ├── 通过 → 床次创建成功(拉布中) → 暂扣层数
  │           │     └── 不通过 → 防超裁：预警/禁止/PC审核
  │           └── 拉布完成 → 提交完成
  │                 ├── 录入实际层数 → 校验通过
  │                 ├── 床次状态→已拉布 → 生成床次号
  │                 └── 锁定实际层数
  │
  └── 缝制挂片（吊挂模块）
        ├── 自动充数模式（默认）
        │     ├── 通用扎卡 → 按CPO交期自动分配
        │     └── 指定扎卡 → 优先绑定→满额顺延
        └── 手动充数模式
              ├── 展示未满额CPO列表
              ├── 自由选择CPO挂片
              ├── CPO满额弹窗提示切换
              └── 随时切回自动模式
```

## 7. Edge Cases & Constraints

### 7.1 并发场景

| 场景 | 处理方式 |
|------|---------|
| 多拉布员同时为同一裁剪任务创建床次 | 实时计算剩余可用层数，先到先得，超出则拒绝 |
| 多拉布员在多裁剪任务中同时引用同一唛架 | 唛架层数分配在裁剪任务创建时即锁定，PAD端仅校验剩余 |
| 同一秒内多次床次提交 | 以数据库记录 ID 为准确保床次号严格递增 |

### 7.2 边界场景

| 场景 | 预期行为 |
|------|---------|
| 唛架计划层数全部分配完毕（剩余=0） | 后续裁剪任务不可选择该唛架 |
| 剩余可用层数恰好等于输入层数 | 校验通过，剩余变为 0，下一床次触发预警 |
| 剩余层数低于预警阈值（如 10 层） | PAD 软提醒："剩余层数不足X层，请注意控制" |
| 输入层数导致剩余为负 | 强控禁止："总裁剪层数已超计划，需PC端审核" |
| 提交实际层数大于暂扣层数 | 校验：实际 ≤（原暂扣+实时剩余）→ 通过；否则失败 |
| 裁剪任务状态为"拉布中"时编辑/删除 | 按钮隐藏，操作禁止 |
| 手动模式下 CPO 全部满额 | 无法继续挂片，需等待新 CPO 或切换生产订单 |
| 自动模式下指定 CPO 满额后无其他 CPO | 无法继续挂片 |

### 7.3 异常场景

| 场景 | 预期行为 |
|------|---------|
| 扫描无效布卷号 | 提示"未找到该布卷号" |
| 输入非正整数组层数 | 提示"请输入有效的正整数层数" |
| 网络中断时提交床次 | 提示提交失败，可重试；暂扣层数保持 |
| PAD端同时操作已被删除的裁剪任务 | 提示"任务不存在或已被删除" |
| 手动充数扫描已满额 CPO 的指定扎卡 | 弹出提示"CPO [编号] 数量已满，请切换其他 CPO" |
