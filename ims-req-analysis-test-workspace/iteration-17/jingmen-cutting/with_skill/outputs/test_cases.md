## 核心流程测试

### 流程1：单CPO裁剪任务→床次创建→拉布完成→扎卡打印→挂片 <!-- F-01 P0 -->

- PC端：创建生产订单（含CPO-A，500件，交期7/1）
  - 进入裁剪任务创建页面
    - 系统显示当前生产订单的CPO列表：CPO-A(500件，交期7/1)
  - 勾选CPO-A，选择唛架MK-001（总计划层数100）
    - 输入分配层数40 → 系统创建裁剪任务CT-001
    - CT-001状态="未开始"，唛架MK-001剩余=60
  - 点击发布 → CT-001状态变为可执行
    - PAD端可见CT-001

- PAD端：拉布员进入CT-001
  - 点击【新增床次】，扫描布卷号FB001
    - 输入拉布层数30 → 系统校验30≤60 → 通过
    - 床次B-01状态="拉布中"，暂扣30层，剩余可用=30
  - 拉布完成，点击【提交完成】
    - 输入实际层数30 → 校验30≤(30+30) → 通过
    - B-01状态="已拉布"，床次号生成（如C-001），锁定30层

- PC端：扎卡打印
  - 打开CT-001的扎卡打印页面
    - 系统自动显示"CPO-A（交期7/1，数量500件）"
    - 扎卡上印刷"CPO-A"，按CPO件数分扎

- PAD端：缝制挂片（自动充数模式）
  - 扫描指定扎卡（CPO-A）
    - 系统自动将裁片充入CPO-A
    - 每扫一张扎卡，CPO-A已充数+1

### 流程2：多CPO合并裁剪→智能合并→床次并发→防超裁 <!-- F-02 P0 -->

- PC端：创建生产订单（含CPO-B/C/D，交期各不相同）
  - 点击【智能合并】
    - 系统将同交期CPO-B(7/5,300)+CPO-C(7/5,200)自动归为一组
    - CPO-D(7/10,400)独立一组
  - 用户选择第一组(B+C)，选择唛架MK-002（总计划层数80）
    - 输入分配层数50 → 创建CT-002，唛架MK-002剩余=30

- PAD端：两个拉布员并发为CT-002创建床次
  - 拉布员A创建床次B-03，输入层数20 → 校验20≤30 → 通过，暂扣20，剩余=10
  - 拉布员B创建床次B-04，输入层数15 → 校验15≤10 → 失败，触发预警"剩余层数不足10层"
  - 拉布员B输入层数10 → 校验10≤10 → 通过，暂扣10，剩余=0

### 流程3：手动充数模式挂片 <!-- F-03 P1 -->

- PAD端：缝制挂片页面，当前生产订单有CPO-A/B/C
  - 员工点击【手动充数】
    - 系统展示所有未满额CPO：CPO-A(剩余200), CPO-B(剩余150), CPO-C(剩余100)
  - 员工选择CPO-B并开始挂片
    - 扫描扎卡 → 裁片绑定到CPO-B
    - CPO-B已充数持续增加，直到剩余=0
  - CPO-B满额时
    - PAD弹出提示："CPO B 数量已满，请切换其他CPO"
    - 员工选择CPO-C → 继续挂片到CPO-C
  - 员工点击【恢复自动充数】
    - 系统按交期重新计算，自动选定交期最近的未满额CPO

## 详细功能点测试

## 裁剪任务模块

### CPO 选择功能

#### 单一CPO勾选创建裁剪任务 <!-- C-01 P0 -->

- admin 登录PC端，进入生产→裁剪任务，点击新建
- 在生产订单颜色基础信息页面，CPO选择框展示当前订单所有CPO
- 勾选单个CPO（CPO-A），系统仅对CPO-A的订单数量生成裁剪任务
- 裁剪任务创建成功，关联CPO="CPO-A"

#### 多CPO勾选创建裁剪任务 <!-- C-02 P0 -->

- admin 登录PC端，进入裁剪任务新建页面
- 勾选CPO-B和CPO-C（不同交期），系统对两个CPO的合并数量生成裁剪任务
- 裁剪任务创建成功，关联CPO="CPO-B, CPO-C"

#### 全选CPO功能 <!-- C-03 P1 -->

- admin 登录PC端，进入裁剪任务新建页面
- 点击CPO选择框的【全选】，系统自动勾选当前生产订单所有CPO
- 所有CPO被选中，全选按钮状态变为"已全选"

#### 智能合并CPO <!-- C-04 P1 -->

- admin 登录PC端，进入裁剪任务新建页面，当前订单有CPO-A(7/1), CPO-B(7/5), CPO-C(7/5), CPO-D(7/10)
- 点击【智能合并】
  - 系统将同交期CPO-B和CPO-C自动归为一组
  - CPO-A和CPO-D各独立一组
  - 弹窗展示分组结果，用户确认后按组创建裁剪任务

### 唛架层数分配与递减

#### 唛架层数分配成功后剩余递减 <!-- V-01 P1 -->

- 唛架MK-001总计划层数=100
- 为CPO-A创建裁剪任务，分配40层 → 创建成功，MK-001剩余可用=60
- 再为CPO-B创建裁剪任务，分配30层 → 创建成功，MK-001剩余可用=30
- 验证：MK-001已分配=70层（40+30），剩余=30层

#### 唛架剩余层数为0时禁止选择 <!-- V-02 P1 -->

- 唛架MK-003总计划层数=50，已全部分配完毕（剩余=0）
- 新建裁剪任务时选择唛架MK-003 → 唛架下拉列表不显示MK-003，或显示为灰色不可选

#### 分配层数超过唛架剩余时拒绝保存 <!-- V-03 P1 -->

- 唛架MK-001剩余可用=10层
- 新建裁剪任务，选择MK-001，输入分配层数20 → 提交
- 系统拒绝保存，提示"分配层数超过唛架剩余可用层数（剩余10层）"

#### 分配层数为0时拒绝保存 <!-- V-04 P2 -->

- 唛架MK-001剩余可用=60层
- 新建裁剪任务，选择MK-001，输入分配层数0 → 提交
- 系统拒绝保存，提示"计划层数必须大于0"

#### 唛架总计划层数边界：恰好等于分配 <!-- V-05 P2 -->

- 唛架MK-004总计划层数=30，剩余可用=30
- 分配层数=30 → 创建成功，MK-004剩余=0

#### 唛架未选择时禁止保存 <!-- V-06 P2 -->

- 新建裁剪任务，填写CPO、计划层数，但不选择唛架
- 点击保存 → 系统提示"请选择唛架"

## 裁剪任务管理

### 顶层任务管理

#### 裁剪任务创建后默认为"未开始"状态 <!-- C-05 P0 -->

- PC端创建裁剪任务CT-001，选择CPO-B，分配唛架层数30
- 保存后，任务列表中CT-001显示状态="未开始"
- 此时可看到【编辑】【删除】按钮

#### 裁剪任务发布后PAD端同步可见 <!-- F-04 P0 -->

- PC端发布CT-001（状态从未开始→可执行）
- PAD端登录，进入裁剪任务列表 → CT-001可见
- 任务信息：关联CPO="CPO-B"，计划层数=30，状态="未开始"

#### "未开始"状态可编辑裁剪任务 <!-- U-01 P1 -->

- CT-001状态="未开始"
- 编辑CT-001，修改计划层数从30改为25
- 保存成功，唛架剩余层数从60变为65（归还5层）
- CT-001计划层数更新为25

#### "未开始"状态可删除裁剪任务 <!-- X-01 P1 -->

- CT-001状态="未开始"，唛架MK-001已分配30层给CT-001
- 删除CT-001 → 删除确认弹窗 → 确认
- CT-001从列表消失，唛架MK-001已分配层数归还，剩余+30

#### "拉布中"状态禁止编辑 <!-- V-07 P1 -->

- CT-001状态="拉布中"（已有床次B-01在拉布中）
- 进入CT-001详情 → 【编辑】按钮隐藏或置灰不可点击

#### "拉布中"状态禁止删除 <!-- V-08 P1 -->

- CT-001状态="拉布中"
- 进入CT-001详情 → 【删除】按钮隐藏或置灰不可点击

#### "已完成"状态禁止新增床次 <!-- V-09 P2 -->

- CT-001状态="已完成"（所有床次已提交，剩余层数=0）
- PAD端进入CT-001 → 【新增床次】按钮隐藏或置灰

## 床次任务与拉布模块

### 床次创建（PAD端）

#### 正常创建床次——输入层数小于剩余 <!-- C-06 P0 -->

- CT-001计划层数=50，剩余可用=50
- PAD端进入CT-001，点击【新增床次】
- 扫描布卷号FB001（有效条码），输入拉布层数20
- 系统校验20≤50 → 通过 → 床次创建成功
- 床次状态="拉布中"，剩余可用=30（暂扣20），床次列表中新增一条记录

#### 输入层数恰好等于剩余可用 <!-- C-07 P1 -->

- CT-001剩余可用=10
- 新增床次，扫描FB002，输入层数10
- 系统校验10≤10 → 通过 → 床次创建成功，剩余可用=0

#### 布卷号扫描无效时提示 <!-- V-10 P2 -->

- PAD端新增床次，扫描无效布卷号条码（系统中不存在）
- 系统提示"未找到该布卷号，请检查后重试"

#### 输入非正整数层数时拒绝 <!-- V-11 P2 -->

- PAD端新增床次，扫描FB001
- 输入层数=-5 → 提示"请输入有效的正整数层数"
- 输入层数=0 → 提示"拉布层数必须大于0"
- 输入层数=abc → 输入框自动过滤，仅允许数字

#### 输入层数超过剩余可用——预警触发 <!-- V-12 P1 -->

- CT-001剩余可用=8层（预警阈值=10）
- 新增床次，输入层数=5（5≤8通过）
  - 系统弹出软提醒："剩余层数不足3层，请注意控制"
- 再新增床次，输入层数=5（5>3不通过）
  - 系统禁止提交："总裁剪层数已超计划，需PC端审核"

#### 输入层数超过剩余可用——强控禁止 <!-- V-13 P1 -->

- CT-001剩余可用=5层
- 新增床次，输入层数=10
- 系统禁止创建，提示"总裁剪层数已超计划，需PC端审核"
- 床次未创建，剩余可用保持=5

#### 并发创建床次——先到先得 <!-- V-14 P2 -->

- CT-001剩余可用=20层
- 拉布员A和拉布员B同时新增床次，均输入层数15
- 拉布员A先提交 → 校验15≤20 → 通过，暂扣15，剩余=5
- 拉布员B提交 → 校验15≤5 → 失败，禁止创建，提示超裁

### 床次提交完成

#### 实际层数等于暂扣层数——提交成功 <!-- C-08 P1 -->

- 床次B-01状态="拉布中"，暂扣层数=20，剩余=30
- 点击【提交完成】，录入实际层数=20
- 校验：20 ≤ (20+30) → 通过
- B-01状态="已拉布"，锁定20层，生成床次号（如C-003）

#### 实际层数小于暂扣层数——归还差额 <!-- V-15 P1 -->

- 床次B-02暂扣层数=20，剩余=30
- 提交完成，录入实际层数=15
- 校验：15 ≤ (20+30) → 通过
- B-02锁定15层，归还5层到剩余可用，剩余变为35

#### 实际层数大于暂扣+实时剩余——提交失败 <!-- V-16 P1 -->

- 床次B-03暂扣层数=20，此时剩余=5
- 提交完成，录入实际层数=30
- 校验：30 > (20+5=25) → 失败
- 提示"实际层数超出可调整范围"，B-03保持拉布中状态，暂扣不变

### 床次号唯一性

#### 床次号在生产订单内严格递增 <!-- V-17 P1 -->

- 生产订单PO-001，裁剪任务CT-A和CT-B
- CT-A提交床次 → 生成C-001（时间10:01）
- CT-B提交床次 → 生成C-002（时间10:05）
- CT-A再提交床次 → 生成C-003（时间10:07）
- 验证：C-001/C-002/C-003在同一PO内全局递增，即使跨裁剪任务

#### 同一秒多次提交——以DB记录ID为准 <!-- V-18 P2 -->

- 两个拉布员在10:15:00同时提交完成
- 床次号按数据库插入顺序分配（C-004, C-005），确保严格递增

#### 主布床次C开头，辅布床次F开头 <!-- V-19 P2 -->

- 主布（cutType=1）裁剪任务提交床次 → 床次号格式：C-001, C-002...
- 辅布（cutType=2）裁剪任务提交床次 → 床次号格式：F-001, F-002...

## 防超裁机制

#### 剩余层数低于阈值——软预警 <!-- V-20 P1 -->

- 剩余可用层数=9（阈值=10）
- 新增床次，输入层数=5 → 校验通过，但弹出提示：
- "剩余层数不足4层，请注意控制"

#### 层数导致剩余为负——强控禁止 <!-- V-21 P1 -->

- 剩余可用层数=3
- 新增床次，输入层数=5
- 系统禁止创建，提示"总裁剪层数已超计划，需PC端审核"
- PC端裁剪任务详情出现【申请超裁】按钮

#### PC端超裁审核——放行 <!-- V-22 P1 -->

- 负责人登录PC端，进入CT-001详情
- 点击【申请超裁】→ 审核通过放行
- 裁剪任务计划层数增加（例如+10），PAD端剩余可用相应增加

#### PC端超裁审核——驳回 <!-- V-23 P2 -->

- 负责人登录PC端，进入CT-001详情
- 点击【申请超裁】→ 驳回
- 裁剪任务计划层数不变，PAD端剩余可用不变
- 拉布员需调整层数后重新提交

## 缝制挂片模块

### 自动充数模式

#### 通用扎卡——按交期自动分配 <!-- F-05 P0 -->

- 生产订单有CPO-A(7/1,500), CPO-C(7/2,200), CPO-B(7/5,300)
- PAD端自动充数模式，扫描通用扎卡
  - 第1-500张：系统自动充入CPO-A（交期最近7/1）
  - CPO-A满额后：第501-700张自动充入CPO-C（交期次之7/2）
  - CPO-C满额后：第701-1000张自动充入CPO-B（7/5）

#### 指定扎卡——优先绑定指定CPO <!-- F-06 P0 -->

- 生产订单有CPO-A(7/1,500), CPO-B(7/5,300)
- PAD端自动充数模式，扫描指定扎卡（印刷CPO-A）
  - 第1-500张：系统优先充入CPO-A（扎卡指定）
  - CPO-A满额后：第501张起自动按交期顺延到CPO-B

#### 指定扎卡——满额后顺延填充 <!-- F-07 P1 -->

- CPO-A已满额（已充500），CPO-B剩余300
- 再扫描一张指定扎卡（印刷CPO-A）
  - 系统自动转换为自由填充模式
  - 裁片充入CPO-B（交期最近的未满额CPO）

### 手动充数模式

#### 进入手动模式——展示未满额CPO列表 <!-- F-08 P1 -->

- PAD端点击【手动充数】
- 系统展示所有未满额CPO列表（按交期排序）：
  - CPO-A（7/1，剩余200件）
  - CPO-C（7/2，剩余100件）
  - CPO-B（7/5，剩余150件）

#### 手动选择CPO——自由挂片 <!-- F-09 P1 -->

- 员工在手动模式下选择CPO-C（交期7/2）
- 扫描扎卡 → 裁片绑定到CPO-C
- 重复挂片直到CPO-C满额

#### CPO满额提示——强制切换 <!-- F-10 P1 -->

- 手动模式下CPO-C剩余数量降至0
- PAD弹出提示："CPO C 数量已满，请切换其他CPO"
- 员工必须选择其他未满额CPO才能继续挂片
- 若员工不切换，系统禁止继续挂片到满额CPO

### 模式切换联动

#### 自动→手动切换——进度保持 <!-- F-11 P1 -->

- 自动充数模式下已为CPO-A充入200件（共500件）
- 员工点击【手动充数】切换
- 系统保存当前进度（CPO-A已充200），提示"已切换至手动充数模式"
- 手动模式下CPO-A剩余显示=300

#### 手动→自动切换——重新计算 <!-- F-12 P1 -->

- 手动模式下员工为CPO-B充入100件
- 点击【恢复自动充数】
- 系统根据所有CPO实时剩余数量和交期顺序重新计算
- 自动选定交期最近的未满额CPO继续充数

#### 手动模式下自由选择跳过交期顺序 <!-- F-13 P2 -->

- 手动模式下CPO列表：CPO-A(7/1,剩200), CPO-C(7/2,剩100), CPO-B(7/5,剩150)
- 员工自由选择CPO-B（交期最远）开始挂片
- 系统允许，不受交期排序限制
