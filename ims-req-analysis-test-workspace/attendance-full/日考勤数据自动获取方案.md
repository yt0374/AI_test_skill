## 关键实现细节

### 1. 日考勤计算触发（核心难点已解决）

```python
def trigger_daily_calculation(page, start_date: str, end_date: str) -> bool:
    """
    触发日考勤计算并等待完成。

    关键交互链：
    1. 点击"日考勤计算"按钮
    2. 弹框中填充起始/结束日期
    3. 点击"确认"
    4. 轮询等待"任务完成"
    5. 关闭弹框
    """
    # Step 1: 点击按钮
    page.locator("button:has-text('日考勤计算')").click()
    page.wait_for_timeout(1500)

    # Step 2: 填充日期（Modal 中前两个 input）
    date_inputs = page.locator(".ant-modal input")
    date_inputs.nth(0).fill(start_date)  # 起始日期
    date_inputs.nth(1).fill(end_date)    # 结束日期

    # Step 3: 点击确认
    page.locator(".ant-modal .ant-btn-primary").click()

    # Step 4: 轮询"任务完成"
    for _ in range(60):  # 最多等 2 分钟
        page.wait_for_timeout(2000)
        modal_text = page.locator(".ant-modal-body").inner_text()
        if "任务完成" in modal_text:
            break

    # Step 5: 关闭弹框
    page.locator(".ant-modal-close").click()
    return True
```

## 注意事项

1. **环境隔离**：员工数据、考勤数据均按环境存储（`_sit.json` / `_uat.json`）
2. **日期格式**：IMS 使用 `yyyy/MM/dd`（斜杠），API 查询可用 `yyyy/MM/dd` 或 `yyyy-MM-dd`
3. **表头偏移**：Ant Design 表格有选择列，td 索引 = th 索引 + 1
4. **任务轮询**：日考勤计算异步执行，需轮询 body 全文中"任务完成"关键字（非仅 modal-body）
5. **数据新鲜度**：dataset 缓存按日期校验，跨天自动重新提取
6. **Excel 导入**：Name 列填工号、Ac-No 列填姓名（与直觉相反）；Machine 列可留空或填 1
7. **Modal 日期输入**：I MS 的 DatePicker 是 readonly，fill() 需加 `force=True`
8. **按钮匹配**：不要用 `button:has-text('查询'), .ant-btn-primary`，`.ant-btn-primary` 可能在弹窗中先匹配到其他按钮

## 已解决问题记录 (2026-07-09)

### 问题 1：日考勤数据提取始终返回 0 条

**根因**：`test_data_extractor.py` 中 `extract_table_rows()` 两个 bug：

1. **行选择器错误**：使用 `.ant-table-tbody tr` 匹配了非数据行（空状态提示行、展开行），数据被污染
2. **列数不匹配**：Ant Design 表格每行 td 比 th 多一列（选择框），导致 `len(cells) == len(headers)` 永远为 False，**所有行被静默丢弃**

**修复**：
- 行选择器改为 `.ant-table-tbody tr.ant-table-row`
- 列偏移：`offset = len(cells) - len(headers)` → `cells = cells[offset:]`

**影响**：修复后日考勤数据从 0 条 → 正常提取，P2/P3 数据驱动测试从 skip 变为 pass。

### 问题 2：查询按钮匹配到错误元素

**根因**：`button:has-text('查询'), .ant-btn-primary` 中 `.ant-btn-primary` 在弹窗存在时先匹配到弹窗按钮（如"确认导入"），导致点击了错误的按钮。

**修复**：改为遍历 visible buttons，匹配 `"查" in text and "询" in text`。

### 问题 3：日考勤计算弹窗日期填充失败

**根因**：IMS DatePicker 是 `<input readonly>`，`fill()` 超时报 "element is not editable"。

**修复**：使用 `fill(value, force=True)` 或在日期默认值正确时不做填充直接确认。

### 问题 4：Excel 导入列名颠倒

**根因**：IMS 模板中 Name 列填的是工号、Ac-No 列填的是姓名，与直觉相反。按直觉填会导致导入预览数据错误、确认按钮 disabled。

**修复**：`ws.cell(column=1, value=emp["工号"])` → Name 列；`ws.cell(column=2, value=emp["姓名"])` → Ac-No 列。

### 问题 5：员工姓名误取为设备登陆号

**根因**：员工档案表中 Ant Design 表格的列结构复杂（61+列），`all_inner_texts()` 按空格分词后索引错位，导致将"设备登陆号"列的数据当作"姓名"。

**修复**：逐列访问 `<td>` 元素，通过表头关键字定位"工号"和"姓名"列索引，+1 偏移适配选择列。

### 问题 6：导入后数据验证失败（误判为导入未持久化）

**现象**：Excel 导入确认按钮点击成功（自然启用 + 自然点击），API 返回 200，但后续查询始终返回 0 条记录，导致误判为"导入未持久化"。

**根因**：导入实际上已成功（数据在 UI 中可见，10 条记录确认入库），问题出在**导入后的验证查询使用了有 bug 的 `extract_table_rows()`**（见问题 1）。由于表头列偏移和行选择器 bug，验证查询永远返回 0，造成"导入失败"的假象。同时缺少去重逻辑，每次都重复导入相同数据。

**修复**：
1. 修复 `extract_table_rows()` 的行选择器和列偏移 bug（见问题 1）
2. 导入前先查询考勤登记，如当天已有数据则跳过导入
3. 导入后使用修复后的提取器验证，而非依赖有 bug 的旧逻辑

**去重判断**：导入前检查当天考勤登记是否已有数据，避免重复导入。

```python
def _has_today_data(page) -> bool:
    """检查考勤登记中是否已有今日数据"""
    page.goto(DETAIL_URL)
    # 点击查询
    for btn in page.locator("button:visible"):
        if "查" in btn.inner_text() and "询" in btn.inner_text():
            btn.click(); break
    page.wait_for_timeout(3000)
    rows = page.locator(".ant-table-tbody tr.ant-table-row")
    return rows.count() > 0
```
