## Assumptions Made

| ID | Assumption | Reason | Risk if Wrong |
|----|-----------|--------|---------------|
| ASM-001 | 定时推送频率为 5 分钟 | 分拣统计一般不需要实时, 5min 是常见报表刷新频率 | 频率过高导致 MQ 压力; 过低导致报表延迟 |
| ASM-002 | 缝制组别在无关联任务单时显示空字符串 | 标准空值处理 | 显示 null 或异常文本 |
| ASM-003 | 订单数 = 按 SKU 维度聚合后的数量值 | 需求描述 "当前 SKU(生产订单+颜色+尺码+CPO)的数量" | 如果是其他聚合维度会不同 |
| ASM-004 | 新 MQ 消息格式完全替代旧格式 (非并存) | 需求为"修正"而非"新增" | 旧格式废弃可能影响其他消费者 |
| ASM-005 | 站位配置按客户维度 (非全局) | 荆门和新基配置不同 | 全局配置会导致客户互相影响 |
| ASM-006 | 分拣人员从 `prod_sortation_task_record_detail` 获取 | 修正逻辑的原意 | 从其他地方获取可能导致不一致 |
| ASM-007 | 报表导出格式为 Excel (.xlsx) | IMS 报表导出通用格式 | 可能是 CSV 或其他 |
| ASM-008 | MQ 消息语言为 JSON | 行业标准 | 可能是 Protobuf 或其他 |
