# 需求规格说明: #4140 分拣统计表 数据修正

> 文档版本: v1.0 | 生成日期: 2026-07-07 | 客户: 荆门鹰美 | 环境: jingmen

---

## 1. Overview + Business Context

### 1.1 业务概述

IMS 分拣统计表 (`prod_sortation_task_record`) 是报表模块中用于统计吊挂分拣系统每日产量的核心报表。当前存在两个严重数据准确性问题:

1. **上架多计**: 上架口同一个 `srcBizCode` 会绑定多个分拣衣架, 导致同一件衣服被统计多次, 产量虚高。
2. **下架漏计**: 下架口的分流阀事件丢失, 导致已完成分拣的衣服未被统计, 产量虚低。

由于上述两个方向性问题同时存在, 报表产量与实际产量对比不准, 可能多也可能少, 无法作为生产管理的可靠依据。

此外, 当前报表缺少「缝制组别」和「订单数」两个维度, 用户无法按缝制小组透视产量, 也无法快速了解每个 SKU 对应的订单数量。

### 1.2 业务目标

- **产量修正**: 基于排重后的 `prod_sortation_task_record_detail` 重新计算产量, 消除重复计数和遗漏
- **字段增强**: 报表增加「缝制组别」(关联缝制任务单) 和「订单数」(按 SKU 聚合)

### 1.3 变更范围

| 系统组件 | 变更内容 |
|---------|---------|
| **sortation 服务** | 从 `prod_sortation_task_record_detail` (已排重) 定时推送 MQ 消息, 站位可配置 |
| **MQ 消息** | 新格式: 日期 + SKU + CPO + 站位 + 人 + 当天产量 |
| **mes-production 服务** | 对接新 MQ, 更新 `prod_sortation_task_record`, 修正人员统计逻辑 |
| **PC-分拣统计表** | 新增字段: 缝制组别、订单数 |

### 1.4 数据流

```
sortation 分拣事件流 (原始)
  │
  ├─ 排重处理
  ▼
prod_sortation_task_record_detail (已排重明细)
  │
  ├─ 定时推送 MQ (新格式: 日期+SKU+CPO+站位+人+产量)
  ▼
mes-production 接收 MQ
  │
  ├─ 更新 prod_sortation_task_record
  ├─ 关联 缝制任务单 → 获取缝制组别
  ├─ 按 SKU 聚合 → 计算订单数
  ▼
PC 端「分拣统计表」展示 (带新字段)
```

## 3. Business Rules

| ID | 规则 | 来源 |
|----|------|------|
| **R001** | 产量统计基于 `prod_sortation_task_record_detail` (排重后明细), 不再基于原始事件流 | CHG-01 |
| **R002** | 上架口同一 srcBizCode 仅计 1 次产量 (多衣架绑定去重) | CHG-02 |
| **R003** | 下架口分流阀事件不可丢失, 必须完整参与统计 | CHG-03 |
| **R004** | MQ 推送频率按配置 (建议不高于每 5 分钟) | 推断 |
| **R005** | MQ 消息包含: 日期(统计日)、SKU、CPO、站位编码、操作人、当天产量数 | CHG-05 |
| **R006** | 推送站位按客户配置: 荆门 = 上架站 + 包装站 + 包装容错站 | CHG-04 |
| **R007** | 缝制组别从缝制任务单获取, SKU 无关联任务单时显示为空或「-」 | CHG-08 |
| **R008** | 订单数 = 按 SKU (生产订单+颜色+尺码+CPO) 聚合的数量 | CHG-09 |
| **R009** | 报表导出包含所有展示字段 (含新增的缝制组别、订单数) | 推断 |
| **R010** | 历史数据不受影响, 仅新数据按新逻辑统计 (除非明确要求修历史) | 推断 |

| 源模块 | 目标模块 | 数据/关系 | 测试关注点 |

## 7. Edge Cases & Constraints

### 7.1 数据边界

| 场景 | 边界条件 | 预期行为 |
|------|---------|---------|
| 同一 srcBizCode 上架 1 次 | 去重逻辑验证 | 产量 = 1 |
| 同一 srcBizCode 上架 N 次 (N>1) | 去重核心场景 | 产量 = 1 (去重后) |
| 分流阀事件 0 条丢失 | 下架完整性 | 产量 = 实际 |
| 分流阀事件部分丢失 | 历史问题 | 修正后不再丢失 |
| 当天 0 产量 | 边缘值 | 报表行存在但产量=0 |
| 大量产量 (10000+) | 大数边界 | 不溢出, 正确显示 |
| MQ 消息重复推送 | 幂等性 | 重复消费不重复统计 |
| MQ 消息延迟到达 | 时序 | 按统计日期写入, 不修改历史 |

### 7.2 字段边界

| 场景 | 边界条件 | 预期行为 |
|------|---------|---------|
| 缝制组别存在 | 正常值 | 正确展示组别名称 |
| 缝制组别不存在 (SKU 无关联任务单) | NULL | 显示空或「-」 |
| 订单数 = 0 | 边缘值 | 显示 0 |
| 订单数极大 | 大数 | 不溢出 |
| 站位值不在配置列表 | 异常 | 不推送 (或记录日志) |

### 7.3 客户差异边界

| 场景 | 荆门 | 通用行为 |
|------|------|---------|
| 干燥房路径的分拣事件 | 存在 (荆门特有) | 确保计入统计 |
| 自动绑定上架 | 是 | 去重逻辑适用 |
| 推送站位置 | 上架/包装/包装容错 | 其他客户可能不同 |
